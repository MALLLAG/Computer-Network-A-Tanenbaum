# 3.2 오류 검출과 오류 정정 (Error Detection and Correction)

### 도입부

2장에서 살펴본 것처럼, 통신 채널은 다양한 특성을 가지고 있습니다. 광섬유(optical fiber)와 같은 일부 채널은 전송 오류가 매우 드물게 발생하여 오류율이 매우 낮습니다. 그러나 무선 링크(wireless links)나 노후화된 가입자 회선(aging local loops)과 같은 다른 채널들은 오류율이 수 자릿수(orders of magnitude) 더 높습니다. 이러한 링크에서는 전송 오류가 일상적입니다. 합리적인 비용이나 성능 저하 없이는 오류를 피할 수 없으므로, 전송 오류는 피할 수 없는 현실이며 이를 다루는 방법을 배워야 합니다.

네트워크 설계자들은 오류를 처리하기 위해 두 가지 기본 전략을 개발했습니다. 두 전략 모두 전송되는 데이터에 **중복 정보(redundant information)** 를 추가합니다.

1. **오류 정정 코드 (Error-Correcting Codes)** : 수신자가 전송된 데이터가 무엇이었는지를 **추론(deduce)** 할 수 있을 만큼 충분한 중복 정보를 포함하는 전략입니다. 오류 정정 코드의 사용을 **FEC (Forward Error Correction, 전방 오류 정정)** 라고도 합니다.
2. **오류 검출 코드 (Error-Detecting Codes)** : 오류가 발생했다는 사실만 감지할 수 있을 정도의 최소한의 중복 정보를 포함하고, 재전송을 요청하는 전략입니다.

#### 각 기법의 적용 분야

- **신뢰성이 높은 채널** (예: 광섬유): 오류 검출 코드를 사용하여 간혹 발견되는 불량 블록만 재전송하는 것이 더 경제적입니다.
- **잡음이 많은 채널** (예: 무선 링크): 각 블록에 중복 정보를 추가하여 수신자가 원래 전송된 블록을 직접 파악할 수 있도록 하는 것이 더 좋습니다. 잡음이 많은 채널에서는 재전송된 데이터도 첫 번째 전송과 마찬가지로 오류가 발생할 가능성이 높기 때문에 FEC를 사용합니다.

#### 오류의 유형

오류의 모델은 크게 두 가지입니다:

1. **고립된 단일 비트 오류 (Isolated single-bit errors)** : 열잡음(thermal noise)의 극단적인 값이 순간적으로 신호를 압도하여 발생합니다.
2. **버스트 오류 (Burst errors)** : 오류가 단독이 아닌 연속적으로 발생하는 경향이 있습니다. 무선 채널에서의 깊은 페이딩(deep fade)이나 유선 채널에서의 순간적 전기 간섭(transient electrical interference) 등 물리적 과정에서 비롯됩니다.

#### 버스트 오류의 장단점

- **장점**: 블록 크기가 1000비트이고 오류율이 비트당 0.001일 때, 오류가 독립적이라면 대부분의 블록에 오류가 포함될 것입니다. 그러나 오류가 100개씩 버스트로 발생한다면, 평균적으로 100개 블록 중 1개만 영향을 받게 됩니다.
- **단점**: 오류가 실제로 발생했을 때 고립된 단일 비트 오류보다 정정하기가 훨씬 어렵습니다.

#### 소거 채널 (Erasure Channel)

때로는 오류의 위치를 알 수 있는 경우가 있습니다. 예를 들어, 물리 계층이 0이나 1의 기대값에서 크게 벗어난 아날로그 신호를 수신하여 해당 비트를 손실된 것으로 선언하는 경우입니다. 이러한 상황을 **소거 채널(erasure channel)** 이라고 합니다. 소거 채널에서는 비트 값을 반전시키는 채널보다 오류를 정정하기가 더 쉽습니다. 비트의 값이 손실되었더라도 최소한 어떤 비트가 오류인지를 알기 때문입니다.

### 3.2.1 오류 정정 코드 (Error-Correcting Codes)

네 가지 오류 정정 코드를 살펴봅니다:

1. **해밍 코드 (Hamming codes)** 
2. **이진 합성곱 코드 (Binary convolutional codes)** 
3. **리드-솔로몬 코드 (Reed-Solomon codes)** 
4. **저밀도 패리티 검사 코드 (Low-Density Parity Check codes, LDPC)** 

이 모든 코드는 전송되는 정보에 **중복성(redundancy)** 을 추가합니다. 프레임은 *m*개의 데이터(메시지) 비트와 *r*개의 중복(검사) 비트로 구성됩니다.

#### 블록 코드, 체계적 코드, 선형 코드

- **블록 코드 (Block code)** : *r*개의 검사 비트가 오직 *m*개의 데이터 비트의 함수로만 계산되는 코드입니다.
- **체계적 코드 (Systematic code)** : *m*개의 데이터 비트가 검사 비트와 함께 직접 전송되는 코드입니다.
- **선형 코드 (Linear code)** : *r*개의 검사 비트가 *m*개의 데이터 비트의 **선형 함수**로 계산되는 코드입니다. 배타적 OR(XOR) 또는 모듈로 2 덧셈이 일반적인 선택입니다.

#### 코드워드와 코드율

블록의 전체 길이를 *n*이라 하면 (즉, *n* = *m* + *r*), 이를 **(n, m) 코드**라고 합니다. 데이터와 검사 비트를 포함하는 *n*-비트 단위를 **코드워드(codeword)** 라고 합니다. **코드율(code rate)** 은 코드워드에서 중복이 아닌 정보의 비율, 즉 *m/n*입니다.

#### 해밍 거리 (Hamming Distance)

두 코드워드가 다른 비트 위치의 수를 **해밍 거리(Hamming distance)** (Hamming, 1950)라고 합니다. 두 코드워드를 **XOR**하고 결과에서 1의 개수를 세면 됩니다.

```
10001001
10110001
--------
00111000  → 해밍 거리 = 3
```

그 의미는, 두 코드워드의 해밍 거리가 *d*이면, 하나를 다른 것으로 변환하기 위해 *d*개의 단일 비트 오류가 필요하다는 것입니다.

#### 코드의 해밍 거리와 오류 검출/정정 능력

- ***d*개의 오류를 확실히 검출**하려면: 거리 **d + 1**인 코드가 필요합니다.
- ***d*개의 오류를 정정**하려면: 거리 **2d + 1**인 코드가 필요합니다.

#### 오류 정정 코드의 간단한 예시

네 개의 유효한 코드워드만 있는 코드를 생각해 봅시다:

```
0000000000, 0000011111, 1111100000, 1111111111
```

이 코드의 거리는 **5**이므로, **이중 오류를 정정**하거나 **4중 오류를 검출**할 수 있습니다. 단, 이중 오류 정정과 4중 오류 검출을 **동시에** 수행할 수는 없습니다.

#### 단일 오류 정정을 위한 검사 비트 수의 하한

*m*개의 메시지 비트와 *r*개의 검사 비트로 모든 단일 오류를 정정할 수 있는 코드를 설계한다고 가정하면, 다음 조건이 성립해야 합니다:

> **(m + r + 1) ≤ 2^r**

주어진 *m*에 대해, 이것은 단일 오류 정정에 필요한 검사 비트 수의 **하한(lower bound)** 을 제공합니다.

#### 해밍 코드 (Hamming Codes)

이 이론적 하한은 실제로 Hamming(1950)이 개발한 방법으로 달성할 수 있습니다.

**해밍 코드에서 비트의 번호 매김:**
- 코드워드의 비트는 왼쪽 끝 비트 1부터 시작하여 오른쪽으로 연속적으로 번호가 매겨집니다.
- **2의 거듭제곱 위치** (1, 2, 4, 8, 16, ...)에 해당하는 비트가 **검사 비트(check bits)** 입니다.
- **나머지 위치** (3, 5, 6, 7, 9, ...)에는 *m*개의 **데이터 비트**가 채워집니다.

**어떤 검사 비트가 어떤 데이터 비트를 검사하는지 알아내는 방법:**
- 위치 *k*에 있는 데이터 비트가 어떤 검사 비트에 의해 검사되는지를 알려면, *k*를 2의 거듭제곱의 합으로 표현합니다.
- 예: 11 = 1 + 2 + 8이므로, 비트 11은 비트 1, 2, 8에 의해 검사됩니다.
- 예: 29 = 1 + 4 + 8 + 16이므로, 비트 29는 비트 1, 4, 8, 16에 의해 검사됩니다.

**디코딩 과정:**

코드워드가 도착하면, 수신자는 수신된 검사 비트의 값을 포함하여 검사 비트 계산을 다시 수행합니다. 검사 비트가 올바르면 각 검사 결과는 **0**이어야 합니다. 검사 결과가 모두 0이 아니면, 오류가 검출된 것입니다.

**오류 신드롬 (Error Syndrome):**

검사 결과들의 집합이 **오류 신드롬(error syndrome)** 을 형성하며, 이것은 오류를 정확히 찾아내어 정정하는 데 사용됩니다. 예를 들어, 신드롬이 0101 (즉, 4 + 1 = 5)이면 **5번째 비트가 오류**라는 것을 의미합니다.

이 구성은 해밍 거리 **3**인 코드를 제공하며, **단일 오류를 정정**하거나 **이중 오류를 검출**할 수 있습니다. 해밍 코드는 **오류 정정 메모리(error-correcting memory)** 에 사용됩니다.

#### 합성곱 코드 (Convolutional Codes)

**합성곱 코드(convolutional code)** 는 블록 코드가 **아닌** 유일한 코드입니다.

**합성곱 코드의 특징:**
- 인코더는 입력 비트의 **시퀀스(sequence)** 를 처리하고 출력 비트의 **시퀀스**를 생성합니다.
- 블록 코드와 달리 자연스러운 메시지 크기나 인코딩 경계가 없습니다.
- 출력은 현재(current) 및 이전(previous) 입력 비트에 의존합니다. 즉, 인코더는 **메모리(memory)** 를 가지고 있습니다.
- 출력이 의존하는 이전 비트의 수를 **제약 길이(constraint length)** 라고 합니다.
- 합성곱 코드는 **코드율(rate)** 과 **제약 길이**로 명세됩니다.

**NASA 합성곱 코드:**

코드율 *r* = 1/2, 제약 길이 *k* = 7인 **NASA 합성곱 코드**는 1977년 보이저(Voyager) 우주 탐사 미션에서 처음 사용된 이후 802.11 등에서 널리 재사용되고 있습니다.

- 왼쪽의 각 입력 비트는 오른쪽에 두 개의 출력 비트를 생성합니다.
- 출력 비트는 입력과 내부 상태(internal state)의 **XOR 합**입니다.
- 내부 상태는 **6개의 메모리 레지스터**에 유지됩니다.
- 다른 비트가 입력될 때마다, 레지스터의 값들이 오른쪽으로 이동합니다.

**디코딩: 비터비 알고리즘 (Viterbi Algorithm)** 

합성곱 코드는 Viterbi가 개발한 알고리즘으로 디코딩됩니다. 비터비 알고리즘은 관찰된 시퀀스를 따라가면서, 각 단계에서 각 가능한 내부 상태에 대해 관찰된 시퀀스를 가장 적은 오류로 생성했을 입력 시퀀스를 추적합니다.

**소프트 판정 디코딩과 하드 판정 디코딩:**

- **소프트 판정 디코딩(soft-decision decoding)** : 비트의 불확실성과 함께 작업하는 접근법입니다. 예를 들어, 0.9V를 "매우 높은 확률로 1"로 취급합니다.
- **하드 판정 디코딩(hard-decision decoding)** : 각 비트가 0인지 1인지 먼저 결정하고 이후에 오류 정정을 수행하는 것입니다.

합성곱 코드는 GSM 이동전화 시스템, 위성 통신, 802.11에서 사용됩니다.

#### 리드-솔로몬 코드 (Reed-Solomon Codes)

**리드-솔로몬 코드(Reed-Solomon code)** 는 **선형 블록 코드**이며, 개별 비트가 아닌 *m*-비트 **심볼(symbols)**  단위로 동작합니다.

**수학적 기반:**

리드-솔로몬 코드는 모든 *n*차 다항식(polynomial)이 *n* + 1개의 점에 의해 유일하게 결정된다는 사실에 기반합니다.

**실제 코드 구조:**

*m*-비트 심볼의 경우, 코드워드는 2^m - 1 심볼 길이입니다. 일반적인 선택은 *m* = 8 (심볼이 바이트)으로, 코드워드는 255바이트 길이가 됩니다. 널리 사용되는 **(255, 233) 코드**는 233개의 데이터 심볼에 32개의 중복 심볼을 추가합니다.

**오류 정정 능력:**

- *m*-비트 심볼에 기반하므로, 단일 비트 오류와 *m*-비트 버스트 오류 모두 하나의 심볼 오류로 처리됩니다.
- 2*t*개의 중복 심볼을 추가하면, 전송된 심볼 중 최대 *t*개의 오류를 정정할 수 있습니다.
- (255, 233) 코드의 경우 최대 **16개의 심볼 오류**, 즉 **최대 128비트의 오류 버스트**를 정정할 수 있습니다.
- 소거(erasure) 모델에서는 최대 **2*t*개의 오류**를 정정할 수 있습니다.

리드-솔로몬 코드는 DSL, 케이블 데이터 전송, 위성 통신, CD, DVD, 블루레이 디스크 등에서 사용됩니다. 종종 합성곱 코드와 결합하여 사용됩니다.

#### 저밀도 패리티 검사 코드 (LDPC - Low-Density Parity Check Codes)

**LDPC (Low-Density Parity Check, 저밀도 패리티 검사) 코드**는 Robert Gallagher가 그의 박사 학위 논문(Gallagher, 1962)에서 발명한 **선형 블록 코드**입니다.

- 각 출력 비트는 입력 비트의 **일부(fraction)** 로부터만 형성됩니다.
- 이는 1의 밀도가 낮은(low density of 1s) 행렬 표현으로 이어지며, 이것이 코드 이름의 유래입니다.
- 수신된 코드워드는 **근사 알고리즘(approximation algorithm)** 으로 디코딩됩니다.
- LDPC 코드는 큰 블록 크기에서 실용적이며, 뛰어난 **오류 정정 능력**을 가지고 있습니다.
- 디지털 비디오 방송 표준, 10 Gbps 이더넷, 전력선 네트워크, 최신 버전의 802.11에서 사용됩니다.

### 3.2.2 오류 검출 코드 (Error-Detecting Codes)

광섬유(fiber)나 고품질 구리선(copper) 위에서는 오류율이 훨씬 낮으므로, **오류 검출과 재전송(retransmission)** 이 간헐적 오류를 처리하는 데 더 효율적입니다.

세 가지 오류 검출 코드를 살펴봅니다:

1. **패리티 (Parity)** 
2. **체크섬 (Checksums)** 
3. **순환 중복 검사 (Cyclic Redundancy Checks, CRCs)** 

#### 패리티 (Parity)

**패리티 비트의 원리:**
- 패리티 비트는 코드워드의 1의 개수가 짝수(even parity)이거나 홀수(odd parity)가 되도록 선택됩니다.
- 예: `1011010`이 짝수 패리티로 전송되면 `10110100`이 됩니다. 홀수 패리티로는 `10110101`이 됩니다.
- 단일 패리티 비트를 가진 코드는 거리가 **2**이며, 모든 단일 비트 오류를 검출할 수 있습니다.

**효율성 비교 (블록 크기 1000비트, 오류율 10^-6):**
- **오류 정정(해밍 코드)** : 1메가비트당 10,000개의 검사 비트 필요
- **오류 검출(패리티)** : 1메가비트당 2,001비트만 필요 (1000블록 × 1패리티비트 + 1번의 재전송 1001비트)

**단일 패리티 비트의 한계:**
- 단일 패리티 비트는 **단일 비트 오류만** 신뢰성 있게 검출할 수 있습니다.
- 긴 버스트 오류에 의해 블록이 심하게 훼손되면, 오류가 검출될 확률은 겨우 **0.5**입니다.

#### 인터리빙 (Interleaving)

단일 비트 오류를 검출하는 코드를 버스트 오류를 검출하는 코드로 변환하는 일반적인 기법입니다.

- *n*비트 너비, *k*비트 높이의 직사각형 행렬로 데이터를 배치합니다.
- *n*개의 열(column) 각각에 대해 패리티 비트를 계산합니다.
- 데이터는 행 단위로 전송하고, 마지막 행에서 *n*개의 패리티 비트를 전송합니다.

**검출 능력:**
- 길이 *n* 이하의 **단일 버스트 오류**를 검출할 수 있습니다.
- 블록이 심하게 훼손된 경우, 잘못된 블록이 수락될 확률은 **2^(-n)** 입니다.

#### 체크섬 (Checksums)

메시지의 데이터 비트에 대한 **누적 합(running sum)** 에 기초한 코드입니다. 체크섬은 보통 합 함수(sum function)의 **보수(complement)** 로서 메시지의 끝에 배치됩니다.

**인터넷 체크섬 (Internet Checksum):**
- IP 프로토콜의 일부로 모든 인터넷 패킷에 사용되는 **16비트 체크섬**입니다.
- 메시지 비트를 **16비트 워드**로 나눈 합입니다.
- **1의 보수(one's complement) 연산**으로 계산됩니다.

**체크섬의 한계:**
- 0 데이터의 삭제 또는 추가를 검출하지 못합니다.
- 메시지의 부분 교환(swapping)을 검출하지 못합니다.

**플레처 체크섬 (Fletcher's Checksum):**
- **위치적 요소(positional component)** 를 포함하여, 데이터 위치 변경에 대해 더 강력한 검출 능력을 제공합니다.

#### 순환 중복 검사 (CRC - Cyclic Redundancy Check)

링크 계층에서 널리 사용되는 **CRC (Cyclic Redundancy Check)** 는 **다항식 코드(polynomial code)** 라고도 합니다.

**기본 원리:**

비트 문자열을 **0과 1의 계수를 가진 다항식의 표현**으로 취급합니다. 예: `110001`은 x^5 + x^4 + x^0을 나타냅니다.

**다항식 산술 (Polynomial Arithmetic):**
- **모듈로 2(modulo 2)** 로 수행됩니다.
- **덧셈과 뺄셈 모두 XOR(배타적 OR)** 과 동일합니다.

**CRC 계산 알고리즘:**

송신자와 수신자는 사전에 **생성 다항식(generator polynomial)** , G(x)에 합의해야 합니다.

1. *r*을 G(x)의 차수라 합니다. 프레임 뒤에 *r*개의 0비트를 추가합니다.
2. G(x)에 대응하는 비트 문자열로 모듈로 2 나눗셈합니다.
3. 나머지를 프레임의 0비트 부분에서 모듈로 2 뺄셈합니다.

**예시: G(x) = x^4 + x + 1, 프레임 = 1101011111**

1. 프레임 뒤에 4개의 0 추가: `11010111110000`
2. `10011`로 모듈로 2 나눗셈 수행
3. 나머지: `1110`
4. 전송 프레임: `11010111111110`

**CRC의 오류 검출 능력:**

1. **단일 비트 오류**: G(x)가 두 개 이상의 항을 포함하면 **모든 단일 비트 오류가 검출**됩니다.
2. **이중 비트 오류**: G(x)가 x^k + 1을 나누지 않는 조건에서 검출됩니다. 예: x^15 + x^14 + 1은 32,768 미만의 어떤 *k*에 대해서도 x^k + 1을 나누지 않습니다.
3. **홀수 개의 비트 오류**: **G(x)의 인수로 x + 1을 포함**시키면, 홀수 개의 반전된 비트를 가진 모든 오류를 검출할 수 있습니다.
4. **버스트 오류**: *r*개의 검사 비트를 가진 다항식 코드는 **길이 r 이하의 모든 버스트 오류를 검출**합니다.
5. **길이 r + 1인 버스트 오류**: 잘못된 프레임이 유효한 것으로 인정될 확률은 **1/2^(r-1)** 입니다.
6. **r + 1보다 긴 버스트**: 검출되지 않을 확률은 **1/2^r**입니다.

#### 국제 표준 다항식

이더넷(Ethernet)의 표준 CRC-32:

> **x^32 + x^26 + x^23 + x^22 + x^16 + x^12 + x^11 + x^10 + x^8 + x^7 + x^5 + x^4 + x^2 + x^1 + 1**

이 다항식은 길이 32 이하의 모든 버스트 및 홀수 개의 비트에 영향을 주는 모든 버스트를 검출합니다. 해밍 거리는 **4**입니다. Castagnoli et al. (1993)과 Koopman (2002)이 철저한 전산 탐색을 통해 해밍 거리 **6**을 가지는 최적의 CRC를 찾았습니다.

CRC는 간단한 **시프트 레지스터 회로(shift register circuits)** 로 하드웨어에서 쉽게 구현됩니다. 사실상 모든 LAN(이더넷, 802.11)과 점대점 링크에서 사용됩니다.
